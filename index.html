Modular Websites

Modular website building is by far the best paradigm for quick development. When done correctly, all pages can look the same yet still be allowed to be edited independently if creative ideas want to be explored. Building modularly should follow the KISS principle (Keep It Simple Stupid). We do not walk around with <head>s in our <body>s, atleast most of us don't. Knowing this, we should always logically separate information, it just werks. If your website cannot follow the 80x24 character width then you shouldnt have a website. God told me 80x24 was the character dimension for the holy people.Javascript was a mistake. It should only be used to bootstrap the only viable web language WASM. Stop using a web browser. Instead use

curl [url] | pandoc -f html -t markdown | mdcat | urlview

Combine these into a shell script for better performance. HTML was sent from God to spread his word in the modern age. Tim Berners-Lee is the second coming of Moses. JS was sent from Satan to ruin the perfect harmony of the internet. Netscape was the original sin.
